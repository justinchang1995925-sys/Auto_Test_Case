# Auto Test Case Skill

`Auto Test Case Skill` 是一个面向高质量测试设计的 Cursor Skill 项目，用于根据多源输入文档自动生成测试点思维导图、测试用例主表和评审追溯附表。

该项目当前核心 Skill 为 `req-testcase-generator`，适用于测试工程师、测试负责人，以及需要参与测试评审的产品和研发同学。

## 项目目标

将分散的需求信息、技术方案、UI 设计和交互说明统一纳入测试设计流程，输出：

- 六维度测试点思维导图
- 测试执行主表
- 评审追溯附表
- 质量审计结果

项目默认仅支持高质量模式，所有输入均按统一高标准执行，不区分快速模式或草稿模式。

## 支持的输入

支持两类维度的输入：

### 1. 文档内容类型

- 需求文档（PRD / BRD）
- 技术方案文档
- UI 设计文档
- 交互文档

### 2. 文档载体格式

- 飞书文档链接
- PDF
- Word（`.docx` / `.doc`）
- TXT / Markdown
- 直接粘贴到对话中的正文

## 输出内容

### 1. 测试点思维导图

基于以下六个维度生成测试点：

- 功能
- 性能
- 稳定性
- 兼容性
- 安全
- 用户体验

### 2. 测试用例主表

用于测试执行，固定字段如下：

| 用例序号 | 优先级（P0-P3） | 测试标题 | 测试类型 | 前置条件 | 操作步骤 | 预期结果 |
|---------|----------------|---------|---------|---------|---------|---------|

### 3. 测试用例附表

用于评审追溯和风险讨论，字段如下：

| 用例序号 | 关联需求/方案条目 | 关联测试点 | 设计技法 | 测试数据 | 风险等级 | 风险评估依据 | 用例状态 | 解除条件 | 备注 |
|---------|------------------|-----------|---------|---------|---------|-------------|---------|---------|-----|

## 核心流程

```text
Phase 0  文档摄入（需求 / 技术方案 / UI / 交互）
Phase 1  需求分析 + 待确认问题清单
Phase 2  测试点思维导图
Phase 3  测试用例主表 + 附表
Phase 4  质量审计
```

完整架构与流程图见：

- `REQ-TESTCASE-SKILL-ARCHITECTURE.md`

## 质量机制

项目内置以下质量约束：

- 风险驱动优先级：P0-P3 与风险等级联动
- 主表 + 附表双输出：兼顾执行与评审
- 反模式库：禁止模糊、不可判定、不可执行的用例写法
- 强制质量审计：输出前复核反模式、追溯链路和高风险路径

### 反模式示例

以下写法会被视为低质量用例并要求改写：

- 预期写“显示正常”
- 一步骤验证 3 件事
- 前置条件写“系统正常”
- 操作步骤写“按要求操作”
- 用例标题只是功能名

## 使用方式

在 Cursor 中使用时，可以直接提供文档并发出类似指令：

- `分析需求`
- `问题已确认：...`
- `生成测试点`
- `测试点已确认`
- `生成主表和附表`
- `给出风险依据`

示例：

```text
根据这份 PRD 和技术方案，生成测试点和测试用例
```

```text
这是 UI 稿和交互说明，请补充体验与兼容性用例
```

```text
测试点已确认，输出主表和附表，并附风险依据
```
